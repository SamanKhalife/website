خوشه‌های Kubernetes

Kubernetes یک خوشه قابل دسترسی بالا از کامپیوترهایی است که به عنوان یک واحد تکمیلی به یکدیگر متصل شده‌اند. انتزاعات در Kubernetes به شما این امکان را می‌دهند که برنامه‌های containerized را به یک خوشه ارسال کنید بدون اینکه به صورت خاص آن‌ها را به ماشین‌های خاص مرتبط کنید. برای استفاده از این مدل جدید از ارسال، برنامه‌ها باید به گونه‌ای بسته بندی شوند که آن‌ها را از میزبان‌های خاص جدا کند: آن‌ها باید containerized شوند. برنامه‌های containerized انعطاف‌پذیرتر و قابل دسترس‌تر از مدل‌های ارسال گذشته هستند، جایی که برنامه‌ها به طور مستقیم بر روی ماشین‌های خاص به عنوان بسته‌ها نصب می‌شوند که به طور عمیق به میزبان تجزیه و تحلیل می‌شوند. Kubernetes توزیع و زمانبندی container‌های برنامه را در یک خوشه به یک روش بهینه‌تر اتوماتیک می‌کند. Kubernetes یک پلتفرم منبع‌باز است و برای استفاده تولید آماده است.

یک خوشه Kubernetes شامل دو نوع منبع است:

  • سیستم کنترل خوشه را هماهنگ می‌کند
  • گره‌ها کارگرانی هستند که برنامه‌ها را اجرا می‌کنند

خلاصه:

  • خوشه Kubernetes
  • Minikube

Kubernetes یک پلتفرم مناسب برای تولید است که اجرای بسته‌ها (container‌ها) را در داخل و بین خوشه‌های کامپیوتری ارزیابی می‌کند.


نمودار خوشه


سیستم کنترل مسئول مدیریت خوشه است. سیستم کنترل همه فعالیت‌های خوشه شما را هماهنگ می‌کند، مانند زمانبندی برنامه‌ها، حفظ حالت مطلوب برنامه‌ها، مقیاس‌پذیری برنامه‌ها و عرضه به‌روزرسانی‌های جدید.

یک گره یک ماشین مجازی یا کامپیوتر فیزیکی است که به عنوان یک ماشین کارگر در یک خوشه Kubernetes خدمت می‌کند. هر گره دارای یک Kubelet است، که یک نماینده برای مدیریت گره و ارتباط با سیستم کنترل Kubernetes است. گره باید همچنین ابزارهایی برای انجام عملیات container‌ها، مانند {{< glossary_tooltip text="containerd" term_id="containerd" >}} یا {{< glossary_tooltip term_id="cri-o" >}} داشته باشد. یک خوشه Kubernetes که بازدید ترافیک را تنظیم می‌کند حداقل سه گره باید داشته باشد چرا که اگر یک گره خاموش شود، هر دو عضو etcd و یک نمونه سیستم کنترل از دست می‌روند، و تکرار قرار می‌گیرد. شما می‌توانید این خطر را با اضافه کردن بیشتر گره‌های کنترل را اصلاح کنید.

سیستم‌های کنترل مدیریت خوشه و گره‌هایی که برای میزبانی برنامه‌های در حال اجرا استفاده می‌شوند.

زمانی که برنامه‌ها را در Kubernetes ارسال می‌کنید، سیستم کنترل را به شروع کانتینرهای برنامه خود می‌گویید. سیستم کنترل کانتینرها را بر روی گره‌های خوشه زمانبندی می‌کند. قسمت‌های سطح گره، مانند kubelet، با استفاده از API Kubernetes که سیستم کنترل آن را فراهم می‌کند، با سیستم کنترل ارتباط برقرار می‌کنند. کاربران پایانی همچنین می‌توانند مستقیماً با API Kubernetes برای تعامل با خوشه استفاده کنند.

یک خوشه Kubernetes می‌تواند بر روی ماشین‌های فیزیکی یا مجازی نصب شود. برای شروع توسعه Kubernetes می‌توانید از Minikube استفاده کنید. Minikube یک پیاده‌سازی سبک Kubernetes است که یک ماشین مجازی را بر روی کامپیوتر شخصی شما ایجاد کرده و یک خوشه ساده را با تنها یک گره راه‌اندازی می‌کند. Minikube برای سیستم‌عامل‌های Linux، macOS و Windows در دسترس است. رابط کاربری CLI Minikube عملیات‌های ابتدایی برای کار با خوشه شما، از جمله شروع، توقف، وضعیت و حذف، را فراهم می‌کند.

اکنون که بیشتر در مورد Kubernetes می‌دانید، به Hello Minikube مراجعه کنید برای امتحان این بر روی کامپیوتر خودتان.
